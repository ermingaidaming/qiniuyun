# 提交合规清单

## 仓库

- [ ] 远端 GitHub/Gitee 仓库创建时间晚于官方开题时间。
- [ ] 提交截止后仓库可公开访问。
- [ ] 仓库内容包含代码、README、Demo 视频链接。
- [ ] 多个独立模块放在同一仓库的不同子目录，例如 `frontend` 和 `backend`。
- [ ] 仓库中不包含密钥、未授权素材、内部资料或隐私数据。

## Commit

- [ ] 所有 commit 时间戳都在所选批次开始与截止时间之内。
- [ ] 从开发开始到截止前保持持续提交。
- [ ] 不在最后一天一次性导入全部代码。
- [ ] 多人组队时，每位队员使用自己的账号提交 commit。
- [ ] commit message 能说明实际改动。

## PR

- [ ] 每个 PR 只做一件事。
- [ ] PR 标题一句话说明新增或修改内容。
- [ ] PR 描述包含功能描述、实现思路、测试方式。
- [ ] PR 描述与实际代码变更一致。
- [ ] 复用过往代码时，在 PR 描述中注明来源。
- [ ] 合并后 `main` 分支可运行。

## README

- [ ] 说明项目议题方向和核心功能。
- [ ] 说明本地启动方式和环境要求。
- [ ] 列明所有第三方库、框架、模型、云服务和素材来源。
- [ ] 说明原创功能部分。
- [ ] 放置可访问、可播放的 Demo 视频链接。

## Demo 视频

- [ ] 使用声音完整讲解。
- [ ] 展示核心模块和主要效果。
- [ ] 覆盖从启动到使用的关键流程。
- [ ] 上传至可访问平台，例如 bilibili 或云盘。
- [ ] README 中的视频链接无需登录即可播放或可按评审要求访问。

## 知识产权与诚信

- [ ] 未抄袭他人代码或技术方案。
- [ ] 第三方开源代码遵守其许可证。
- [ ] 未提交未获授权的单位知识产权内容。
- [ ] 对 AI 生成、第三方服务、旧代码复用做了必要说明。
