# Demo 视频脚本

建议控制在 4 到 6 分钟，必须有声音讲解。

## 1. 项目开场（30 秒）

- 说明项目名称：AI 小说转剧本工具。
- 说明选题方向：基于大语言模型的 AI 核心应用研发。
- 说明目标用户：需要将长篇网文/IP 小说批量转化为剧本格式的内容创作者和制片方。
- 一句话概括核心能力：上传小说→自动构建因果图→AI 改写→幻觉校正→输出结构化剧本。

## 2. 运行方式（30 秒）

- 展示 README 中的本地启动命令。
- 启动后端 FastAPI 服务，展示 Swagger 文档页（/docs）。
- 启动前端 Next.js 服务，打开浏览器访问本地页面。

## 3. 核心功能演示（3 分钟）

### 3.1 小说上传与解析（45 秒）

- 点击上传按钮，选择一本小说 TXT 文件。
- 展示章节自动解析结果（章节列表、字数统计）。
- 点击任一章节查看原文内容。

### 3.2 CPC 因果情节图构建（45 秒）

- 点击"构建因果图"按钮，触发 CPC 分析。
- 展示 DAG 可视化图：事件节点、因果关系边、时间序。
- 说明贪婪循环破除算法的作用（简要提及循环→DAG 的转化）。

### 3.3 R2 滑动窗口改写（45 秒）

- 点击"开始改写"，触发 R2 扫描。
- 展示改写进度（窗口滑动动画）。
- 对比原文片段与改写后的剧本片段。
- 指出关键改写点（叙述→动作描述、心理活动→括号提示）。

### 3.4 HAR 幻觉校正（30 秒）

- 点击"校正检查"，触发 HAR 自校正。
- 展示检测到的幻觉示例（如果有）及修正建议。
- 展示校正报告：检测项数量、修正数量、最终置信度。

### 3.5 ScreenYAML 生成与导出（15 秒）

- 展示最终生成的 ScreenYAML 结构（动作、角色、对话、括号）。
- 点击导出 Word/PDF，下载文件。
- 打开导出文件展示格式效果。

## 4. 技术架构说明（30 秒）

- 展示架构图（可用 PPT 或绘图工具）。
- 前端：Next.js + TypeScript + TailwindCSS。
- 后端：Python FastAPI。
- 算法核心：CPC 贪婪循环破除 + R2 滑动窗口重写 + HAR 向量 RAG 自校正。
- 输出：符合 Draft 2026-01 模式的 ScreenYAML。
